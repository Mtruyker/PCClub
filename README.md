# PC Club Admin

WPF-приложение для администрирования компьютерного клуба. В проекте есть экраны для управления компьютерами, клиентами, сессиями, тарифами и статистикой. Данные хранятся в SQLite-базе `pcclub.db` в папке локальных данных пользователя.

## Что было изменено

### Окно сессий

- Добавлен `NullToVisibilityConverter` в `WpfApp8/Infrastructure/NullToVisibilityConverter.cs`.
- В `WpfApp8/Sessions.xaml` кнопка `Завершить` теперь показывается только для активных сессий.
- Если у сессии заполнено `EndTime`, кнопка скрывается через привязку:

```xml
Visibility="{Binding EndTime, Converter={StaticResource NullToVisibilityConverter}}"
```

Это убирает ситуацию, когда у уже закрытой сессии всё ещё висит кнопка завершения.

### Адаптивная разметка экранов

Обновлены основные XAML-экраны:

- `WpfApp8/clients.xaml`
- `WpfApp8/computer.xaml`
- `WpfApp8/Tariffs.xaml`
- `WpfApp8/Sessions.xaml`

Изменения в разметке:

- корневые `StackPanel` заменены на `Grid`;
- верхние панели занимают место по содержимому;
- таблицы занимают всё оставшееся пространство окна;
- убрана фиксированная высота таблиц `Height="400"`;
- колонки `DataGrid` получили `Auto`, `*` и `MinWidth`, чтобы интерфейс лучше подстраивался под содержимое;
- колонки действий расширены, чтобы кнопки `Редактировать`, `Удалить` и `Завершить` не обрезались;
- для таблицы сессий оставлена горизонтальная прокрутка, потому что в ней много колонок.

### Экран статистики

- `WpfApp8/Statistics.xaml` переведён на адаптивную `Grid`-разметку.
- Фильтры статистики теперь переносятся по ширине окна через `WrapPanel`.
- Заглушки во вкладках заменены на реальные таблицы:
  - доходы по дням;
  - распределение сессий по компьютерам;
  - популярные тарифы.
- Для этих таблиц добавлены коллекции `DailyRevenue`, `ComputerDistribution` и `PopularTariffs` во `StatisticsViewModel`.
- Добавлен простой класс `StatisticItem` для отображения строк статистики.

### Модель компьютера

- Из модели `Computer` убрана зависимость от WPF-типа `Brush`.
- Цвет статуса теперь определяется в интерфейсе через `ComputerStatusBrushConverter`.
- `Computer` хранит только данные компьютера и вычисляемый текстовый статус.

### Диалоги

- Добавлен `IDialogService` для предупреждений и подтверждений.
- Добавлена реализация `MessageBoxDialogService`, которая использует стандартный WPF `MessageBox`.
- `ClientsViewModel`, `ComputersViewModel`, `SessionsViewModel` и `TariffsViewModel` больше не вызывают `MessageBox.Show` напрямую.
- ViewModel теперь зависят от интерфейса диалогов, поэтому их проще тестировать и менять без переписывания UI-логики.

### Демо-данные

- Демо-данные вынесены из ViewModel в `DemoDataSeeder`.
- `ClientsViewModel`, `ComputersViewModel`, `SessionsViewModel` и `TariffsViewModel` больше не создают стартовые записи сами.
- `DemoDataSeeder.SeedIfNeeded()` запускается при старте приложения после `DatabaseInitializer.Initialize()`.
- Инициализация демо-данных теперь собрана в одном месте и не смешана с логикой экранов.
- Набор стартовых данных расширен: добавлены дополнительные клиенты, компьютеры, VIP/турнирные тарифы и история сессий за разные дни.
- Активные демо-сессии согласованы со статусом занятости компьютеров.
- Seeder проверяет существующие записи перед добавлением, поэтому повторный запуск приложения не создаёт дубликаты демо-данных.
